Graph Algorithm Time and Space Complexity Analysis

The performance of graph algorithms depends on the graph's representation (adjacency matrix vs. adjacency list) and properties (directed, weighted, etc.).

Graph Representation Complexities

Operation Adjacency Matrix Adjacency List Notes
Storage O(V²) O(V + E) List is better for sparse graphs
Add Edge O(1) O(1) Both are efficient
Remove Edge O(1) O(E) Matrix is better for edge updates
Find Edge O(1) O(V) Matrix provides direct access

Algorithm Time Complexities

Algorithm Time (Matrix) Time (List) Space Notes
BFS O(V²) O(V + E) O(V) Level-order traversal
DFS O(V²) O(V + E) O(V) Recursive stack space
Dijkstra's O(V²) O((V + E) log V) O(V) With binary heap
Bellman-Ford O(V³) O(VE) O(V) Handles negative weights
Prim's MST O(V²) O((V + E) log V) O(V) Minimum spanning tree
Kruskal's MST O(E log E) O(E log E) O(V) Using Union-Find

Important Notes

  • V = number of vertices, E = number of edges
  • For sparse graphs (E ≈ V), adjacency list is more efficient
  • For dense graphs (E ≈ V²), both representations are comparable
  • Space complexity often includes the graph representation
